# 宝宝魔法键盘 · 国内发布指南

目标：把 `index.html` 发布到线上，让**国内的小红书用户**能流畅打开。

> 已为你准备好部署入口文件 `index.html`（内容和 `baby-play.html` 完全一样，
> 静态托管平台默认要求入口文件叫这个名字）。以后改内容时改 `baby-play.html` 后，
> 再复制一份覆盖 `index.html` 即可。

---

## 一、为什么不用 GitHub Pages / Vercel / Netlify

这些国外免费托管在国内**经常很慢甚至打不开**，给小红书国内用户用体验很差，不推荐。
下面的方案都是国内能正常访问的。

---

## 二、主方案：腾讯云 EdgeOne Pages（最推荐）

优点：**免费、国内访问快、送的子域名免备案、几分钟搞定**。

### 步骤

1. 打开 https://edgeone.ai/pages 或腾讯云控制台搜索「EdgeOne Pages」，用微信/QQ 登录。
2. 点「创建项目」→ 选择「直接上传」（或叫「上传部署 / Upload）」。
3. 把本文件夹里的 **`index.html`** 拖进去上传。
   - 如果它要求上传整个文件夹，就把 `index.html` 单独放一个文件夹再上传该文件夹。
4. 部署完成后会得到一个网址，形如 `https://xxxx.edgeone.app`。
5. 用手机浏览器打开这个网址测试一下，能正常出现画面和声音即成功。

> 注：EdgeOne Pages 也支持「连接 Git 仓库自动部署」，但你这是单文件，**直接上传最省事**。

---

## 三、备用方案 A：Gitee Pages（码云）

- 网址：https://gitee.com
- 需要**实名认证**（上传身份证，开通 Pages 服务必需）。
- 步骤：注册 → 新建仓库（设为公开）→ 上传 `index.html` → 仓库「服务」菜单里开启「Gitee Pages」→ 得到访问网址。
- 缺点：实名审核、Pages 部署偶尔要等审核，没 EdgeOne 快。

## 四、备用方案 B：阿里云 OSS / 腾讯云 COS 静态网站托管

- 适合你**已有云账号**、或将来想用自己好记的域名。
- 把 `index.html` 上传到一个 Bucket，开启「静态网站托管」，入口文件设为 `index.html`。
- 用 OSS/COS 默认域名能访问；**绑定自己的域名则必须先备案**（约 1–3 周）。

---

## 五、想要好记的网址（可选，需备案）

- 免费子域名（如 `xxxx.edgeone.app`）够用，但不好记。
- 想要 `baobao.com` 这种：需要**买域名 + ICP 备案**（备案约 1–3 周，必须用国内服务器/CDN）。
- 建议：**先用免费子域名上线**，等火了再考虑买域名备案。

## 六、关键：怎么在小红书让别人访问？

⚠️ 小红书的**笔记正文、评论、个人简介里的网址都不能直接点击**。所以：

### 推荐做法：发二维码图片

1. 拿到上线网址后，用任意「网址转二维码」工具生成二维码图片，例如：
   - 草料二维码：https://cli.im
   - 或微信里直接「网址生成二维码」的小程序
2. 把二维码**保存成图片**，作为小红书笔记的配图发出去。
3. 文案里写一句：「长按图片识别二维码 / 截图用微信扫一扫，就能给宝宝玩啦～」

### 备选做法：让用户手动打开

- 在笔记里写出完整网址，引导：「复制网址到浏览器打开」。
- 把网址放进**个人简介**，引导大家「主页简介里有链接，复制到浏览器打开」。

> 小提示：网址越短越好记，截图扫码体验最好。

## 七、上线后自测清单

- [ ] 手机浏览器能打开，画面正常显示。
- [ ] 触摸屏幕有动画 + 声音（首次需点一下才会出声，浏览器策略所致）。
- [ ] 横屏、不同手机都试一下。
- [ ] 二维码扫码能正常跳转。
